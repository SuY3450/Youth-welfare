# 🤖 LLM RAG 파이프라인 (청년 복지 정책 추천)

## 📌 개요

사용자가 앱에서 입력한 정보(나이, 지역, 소득, 취업상태 등)를 기반으로 **465건의 청년 복지 정책** 중에서 가장 적합한 정책을 자동으로 추천하는 AI 파이프라인입니다.

---

## 🔧 기술 스택

| 구분 | 기술 |
|------|------|
| LLM | Google Gemini 2.5 Flash (개발) → Claude (최종 데모) |
| 프레임워크 | LangChain |
| 벡터 DB | ChromaDB |
| 임베딩 모델 | `jhgan/ko-sroberta-multitask` (한국어 특화) |
| 언어 | Python |

---

## 🏗️ 전체 아키텍처

```
GitHub → 465건 데이터 자동 다운로드
    ↓
sub_region / amount 자동 보완 (raw_text 파싱)
    ↓
ChromaDB 벡터 저장
    ↓
쿼리 확장 (동의어) + 다중 쿼리 (4가지 관점)
    ↓
하이브리드 검색 (카테고리 매핑 + 지역 필터 + MMR)
    ↓
Reasoning (나이/지역/소득/취업 → 파이썬 코드로 직접 계산)
    ↓
중복수혜 체크 (conflict_checker)
    ↓
Gemini (raw_text 포함 프롬프트 → 최종 추천 + 이유)
    ↓
dict 반환 → 백엔드 API → 앱 화면
```

---

## 📋 단계별 상세 설명

### 1단계 — 데이터 로드 + 자동 보완

서버 시작 시 GitHub에서 최신 `clean_final.json`을 자동으로 다운로드합니다.

데이터의 `sub_region`(시/군/구)과 `amount`(지원 금액)가 비어있는 경우, `raw_text`(정책 원문)에서 자동으로 추출하여 보완합니다.

```
sub_region 보완: raw_text에 "평택시" 포함 → sub_region = "평택시"
amount 보완:     raw_text에 "월 최대 20만원" 포함 → amount = "월 최대 20만원"
```

---

### 2단계 — 쿼리 확장 + 다중 쿼리 검색

사용자 입력을 검색에 유리한 문장으로 변환합니다.

**쿼리 확장 (Query Expansion):**

동의어 사전을 활용하여 검색어를 풍부하게 만듭니다.

```
입력: "근무", "월세 자취"
확장: "근무 재직 근로자 직장인 월세 임차 자취 원룸"
```

**다중 쿼리 (Multi-Query):**

같은 사용자를 4가지 관점에서 검색합니다.

| 관점 | 쿼리 예시 |
|------|----------|
| 사용자 상황 | "27세 청년 서울 거주 근무 월세 자취 주거 지원" |
| 혜택 유형 | "주거 월세 자취 지원금 수당 청년 서울" |
| 자격 조건 | "만 27세 서울 청년 주거 혜택" |
| 취업상태 | "근무 청년 서울 주거 지원" |

---

### 3단계 — 하이브리드 검색 (RAG)

ChromaDB에서 의미적으로 유사한 정책을 검색합니다.

**벡터 유사도 검색:**

임베딩 모델(`ko-sroberta-multitask`)이 텍스트를 768차원 벡터로 변환하고, ChromaDB가 L2 거리 기반으로 유사한 정책을 찾습니다.

```
사용자 쿼리 → 벡터 [0.23, -0.15, 0.87, ...]
정책 텍스트 → 벡터 [0.21, -0.13, 0.85, ...]
→ 거리 가까움 = 의미 유사
```

**카테고리 매핑:**

앱의 대분류를 데이터의 세부 분류로 매핑합니다.

```
앱에서 "주거" 선택
→ 데이터에서 "주택 및 거주지", "전월세 및 주거급여 지원", "기숙사" 검색
```

**MMR (Maximum Marginal Relevance):**

비슷한 정책끼리 중복되지 않고 다양한 정책이 추천되도록 합니다.

```
일반 검색: 청년월세A, 청년월세B, 청년월세C (중복)
MMR 검색:  청년월세A, 임대주택B, 보증금지원C (다양)
```

---

### 4단계 — Reasoning (자격 조건 판단)

검색된 정책마다 **파이썬 코드로 직접 계산**합니다 (LLM 아님).

4가지 조건을 순서대로 판단합니다:

| 조건 | 만점 | 조건 미상 시 | 판단 방법 |
|------|------|------------|----------|
| 나이 | 20점 | 10점 (절반) | `age_min ≤ 사용자 나이 ≤ age_max` |
| 지역 | 20점 | - | 정책 지역 = 사용자 지역 or "전국" |
| 소득 | 15점 | 7점 (절반) | 사용자 소득 ≤ 중위소득 기준 % |
| 취업상태 | 15점 | 15점 (무관) | 정책 요구 = 사용자 상태 |
| 의미 유사도 | 30점 | - | `유사도% × 30 / 100` |
| **합계** | **100점** | | |

**예시:**

```
[2026년 청년월세 지원사업] 판단:

① 나이:     조건 미상 → 통과 (+10점)
② 지역:     "전국" → 사용자 서울 → ✅ 통과 (+20점)
③ 소득:     조건 미상 → 통과 (+7점)
④ 취업상태:  무관 → ✅ 통과 (+15점)
⑤ 유사도:   1.7% × 30 / 100 = 0.5점

합계: 10 + 20 + 7 + 15 + 0.5 = 52.5%
```

4가지 조건 **하나라도 탈락하면 부적합**, 전부 통과하면 적합 판정됩니다.

---

### 5단계 — 중복수혜 체크 (conflict_checker)

동시에 받을 수 없는 정책 조합을 자동 감지합니다.

**두 가지 방식으로 감지:**

| 방식 | 설명 | 예시 |
|------|------|------|
| 정책명 기반 | 구체적 정책 간 충돌 테이블 | "청년월세 지원" ↔ "국토부 청년월세" |
| 유형 기반 | 같은 유형 자동 감지 | 월세 지원 2개 → 1개만 가능 |

**처리 방법:**

충돌 시 적합도 높은 정책을 유지하고, 낮은 쪽에 경고를 표시합니다.

```
적합 정책 14건
    ↓ 충돌 감지 3건
    ↓ 적합도 높은 것 유지
최적 조합 11건
```

---

### 6단계 — Gemini 최종 분석

최적 조합 정책들을 Gemini LLM에게 전달하여 **우선순위 + 추천 이유**를 생성합니다.

**프롬프트에 포함되는 정보:**

```
[1] 2026년 청년월세 지원사업 (적합도 52.5%)
  분류: 주거
  금액: 월 최대 20만원
  지역: 전국
  조건 판단: 나이 ⚠️ / 지역 ✅ / 소득 ⚠️ / 취업상태 ✅
  정책 내용: 만 19세~34세 무주택 청년으로서 임차보증금 5천만원... (raw_text 500자)
```

**핵심 포인트:**

- `raw_text`(정책 원문)를 프롬프트에 포함하여 실제 자격조건, 지원금액, 대상 지역을 Gemini가 직접 읽고 판단합니다.
- Gemini는 조건 판단을 하지 않습니다. 이미 Reasoning에서 판단 완료된 결과를 받아서 **"어떤 게 제일 좋은지" + "왜 좋은지"** 만 생성합니다.

---

### 7단계 — 결과 반환

최종 결과를 dict로 반환하여 백엔드 API에서 프론트엔드로 전달합니다.

```json
{
  "results": [
    {
      "name": "2026년 청년월세 지원사업",
      "priority": 1,
      "fit_score": "52.5%",
      "amount": "월 최대 20만원",
      "region": "전국",
      "reason": "서울에 거주하는 무주택 월세 청년에게 월세를 직접 지원"
    }
  ],
  "top_recommendation": "2026년 청년월세 지원사업",
  "total_monthly": "월 최대 20만원",
  "summary": "서울 서초구 거주 27세 청년에게 월세 지원 정책을 추천합니다."
}
```
